# Split-K (Flash-Decoding) for B1 H2 S40000 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.
> **Plan doc is NOT git-committed** (project rule). Code commits within tasks DO happen (TDD).

**Goal:** Add a two-pass split-K (flash-decoding) path so the B1 H2 S40000 d64 workload fills the
GPU's CUs, targeting +20-30% TFLOPS over the single-pass baseline (336 → ~400+).

**Architecture:** Pass 1 = the existing per-block forward pass under a new `IsSplit` template
parameter, walking a KV sub-range and writing fp32 partial output `O_g` + natural-log `LSE_g` to
scratch. Pass 2 = a new lightweight combine kernel that reweights the G partials per (batch, head,
M-tile) by softmax(LSE_g) and writes the final bf16 O. Test-first: oracle → red test → green kernel.

**Tech Stack:** HIP / gfx942 (MI300X, CDNA3), C++17, GoogleTest, OpenMP host refs, rocprof-compute
+ rocprofv3 + amdgpu-asm-analyzer for the profiling/ISA gates.

**Spec:** `docs/superpowers/specs/2026-06-02-splitk-s40000-design.md` (read it first — combine math
base-pinning in §2, scratch layout in §4, gate definitions in §7 are load-bearing).

## Task 2: `cpu_ref_combine` CPU oracle

**Files:**
- Create: `src/components_ref/cpu_ref_combine.hpp`, `src/components_ref/cpu_ref_combine.cpp`

- [ ] **Step 1: Declare the oracle** (`cpu_ref_combine.hpp`):

```cpp
// CPU oracle for the split-K combine. Inputs are G partials per row:
//   o_part[g][d]  : normalized fp32 partial output (range g), d in [0,64)
//   lse_part[g]   : natural-log LSE of range g (-INFINITY if range empty/masked)
// Output o_final[d] = Σ_g w_g * o_part[g][d], w_g = exp(lse_part[g]-L*),
//   L* = ln(Σ_g exp(lse_part[g]-M)) + M, M = max_g lse_part[g].
// All -inf partials contribute 0. If ALL are -inf, o_final = 0.
void cpu_ref_combine(int G, int Dlog,
                 const float* o_part,   // [G*Dlog]
                 const float* lse_part, // [G]
                 float* o_final);       // [Dlog]
```

- [ ] **Step 2: Implement** (`cpu_ref_combine.cpp`) — natural-e domain, numerically stable:

```cpp
#include "components_ref/cpu_ref_combine.hpp"
#include <cmath>
#include <vector>
void cpu_ref_combine(int G, int Dlog, const float* o_part,
                 const float* lse_part, float* o_final) {
    float M = -INFINITY;
    for (int g = 0; g < G; ++g) if (lse_part[g] > M) M = lse_part[g];
    for (int d = 0; d < Dlog; ++d) o_final[d] = 0.0f;
    if (M == -INFINITY) return;             // all empty/masked -> zeros
    double denom = 0.0;
    std::vector<float> w(G);
    for (int g = 0; g < G; ++g) {
        w[g] = (lse_part[g] == -INFINITY) ? 0.0f : expf(lse_part[g] - M);
        denom += w[g];
    }
    float inv = (denom > 0.0) ? (float)(1.0/denom) : 0.0f;
    for (int g = 0; g < G; ++g) {
        float wg = w[g] * inv;              // normalized weight, Σ=1
        if (wg == 0.0f) continue;
        for (int d = 0; d < Dlog; ++d) o_final[d] += wg * o_part[g*Dlog + d];
    }
}
```
